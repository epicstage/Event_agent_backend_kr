## Post-Event Report (PER) 정의
이벤트의 상세 정보와 활동을 기록한 보고서. 시간이 지남에 따라 PER 모음은 반복 이벤트의 완전한 역사를 제공합니다.

## 작성 주체
- 주 이벤트 주최자
- 해당 이벤트의 공급업체와 협력하여 작성
- 사용된 각 venue 및 시설에 제출

## PER 활용 목적
1. 향후 RFP에 첨부하여 과거 실적 증명
2. 반복 이벤트의 역사적 기록
3. 성과 분석 및 개선점 도출

## 주요 섹션

### SECTION V: FOOD & BEVERAGE INFORMATION
```
Were food & beverage (F&B) functions included in the event? Yes/No

Attendance at Largest F&B Function: __________

What type of F&B function was the largest in attendance?
- Break/Continental
- Breakfast
- Lunch
- Reception
- Dinner
- Other: _____________________

What type of F&B function was the largest revenue producer?
(Same options as above)
```

### Room Block 섹션
```
% Contracted (when contracted):
  "Actual pick-up" DIVIDED BY "Contracted block when contracted" × 100 = _____%

% Sold (of final block):
  "Actual pick-up" DIVIDED BY "Final Block" × 100 = _____%

% to peak:
  "Actual pick-up" DIVIDED BY peak night pick-up × 100 = _____%

Room Block Comments: __________
```

## 핵심 메트릭

| 항목 | 설명 |
|------|------|
| Contracted Block | 계약 시 예약된 객실 수 |
| Final Block | 최종 객실 블록 |
| Actual Pick-up | 실제 사용된 객실 수 |
| Peak Night | 가장 많이 사용된 날 |
| F&B Revenue | 식음료 매출 |
| Attendance | 참석자 수 |

## 권장 사항
1. 일부 정보는 필수 입력
2. 최신 PER은 향후 RFP와 함께 제출
3. 각 venue/시설별로 별도 제출
