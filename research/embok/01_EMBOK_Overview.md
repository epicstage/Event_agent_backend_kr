## EMBOK이란?
Event Management Body of Knowledge (EMBOK)는 이벤트 관리의 지식과 프로세스를 수집하고 연구하기 위한 구조를 제공하는 프레임워크입니다.

## EMBOK의 4대 구성요소 (Facets)

### 1. Phases (단계)
이벤트 관리의 순차적 특성을 보여주며, 시간의 중요성을 강조합니다.

| Phase | 설명 |
|-------|------|
| **Initiation** | 시작/착수 단계 |
| **Planning** | 기획 단계 |
| **Implementation** | 실행 단계 |
| **Event** | 이벤트 당일 |
| **Closure** | 마무리 단계 |

### 2. Processes (프로세스)
순차적이면서도 반복적인 시스템. 포괄적인 행동 과정과 변화하는 이벤트 특성에 대한 동적 접근을 촉진합니다.

| Process | 설명 |
|---------|------|
| **Assessment** | 평가 |
| **Selection** | 선택 |
| **Monitoring** | 모니터링 |
| **Communication** | 커뮤니케이션 |
| **Documentation** | 문서화 |

### 3. Core Values (핵심 가치)
모든 요소, 단계, 프로세스에 관한 모든 결정에 스며들어야 하는 원칙. 성공적이고 지속 가능한 결과를 보장합니다.

| Core Value | 설명 |
|------------|------|
| **Continuous Improvement** | 지속적 개선 |
| **Creativity** | 창의성 |
| **Ethics** | 윤리 |
| **Integration** | 통합 |
| **Strategic Thinking** | 전략적 사고 |

### 4. Knowledge Domains (지식 영역)
이벤트 관리의 주요 활동 영역 또는 기능을 나타냅니다.

| Domain | 설명 |
|--------|------|
| **Administration** | 관리/행정 |
| **Design** | 디자인 |
| **Marketing** | 마케팅 |
| **Operations** | 운영 |
| **Risk** | 리스크 관리 |

## EMBOK 정의
> "Event management is the process by which an event is planned, prepared, and produced. As with any other form of management, it encompasses the assessment, definition, acquisition, allocation, direction, control, and analysis of time, finances, people, products, services, and other resources to achieve objectives."

- 이벤트 프로젝트 관리
