## 1. Phases (단계)

이벤트 관리의 순차적 특성을 보여주며, 시간의 중요성을 강조합니다.

### 5가지 단계

```
┌─────────────┐   ┌──────────┐   ┌────────────────┐   ┌─────────┐   ┌──────────┐
│ INITIATION  │ → │ PLANNING │ → │ IMPLEMENTATION │ → │  EVENT  │ → │ CLOSURE  │
│   착수      │   │   기획   │   │     실행       │   │ 이벤트  │   │  마무리  │
└─────────────┘   └──────────┘   └────────────────┘   └─────────┘   └──────────┘
```

### 1.1 Initiation (착수)

**정의**: 이벤트 프로젝트의 시작점

**주요 활동**:
- 이벤트 목적 및 목표 정의
- 타당성 검토 (Feasibility Study)
- 이해관계자 식별
- 초기 예산 및 일정 수립
- Go/No-Go 의사결정

**산출물**:
- 프로젝트 헌장 (Project Charter)
- 초기 범위 정의서

### 1.2 Planning (기획)

**정의**: 상세 계획 수립 단계

**주요 활동**:
- 상세 예산 편성
- 일정 개발 (Master Schedule)
- 벤더/공급업체 선정
- 리스크 분석
- 마케팅 전략 수립
- 운영 계획 작성

**산출물**:
- 이벤트 기획서
- 예산서
- 마스터 일정표
- 리스크 관리 계획

### 1.3 Implementation (실행)

**정의**: 계획을 실제로 실행하는 가장 긴 단계

**주요 활동**:
- 계약 체결 및 관리
- 등록 시스템 운영
- 마케팅 캠페인 실행
- 물류 준비
- 리허설 및 테스트
- 현장 셋업

**특징**:
- 일반적으로 가장 긴 기간
- 이 단계까지 Go/No-Go 결정 가능

### 1.4 Event (이벤트)

**정의**: 이벤트 당일 운영

**주요 활동**:
- 참석자 등록/체크인
- 프로그램 진행
- 현장 문제 해결
- 실시간 모니터링
- 비상 상황 대응

**특징**:
- 짧은 기간이지만 가장 집중적
- "Plan B" 즉시 실행 준비 필요
- 이 단계에서는 취소만 가능 (No-Go 불가)

**비상 시 조치**:
- 팀원 즉시 통보
- 이해관계자, 참석자, 참가자 안내
- 대체 계획 (Plan B) 실행

### 1.5 Closure (마무리)

**정의**: 이벤트 종료 및 평가

**4가지 구성요소**:
1. **Event Shutdown** - 철거, 반납, 정산
2. **Feedback & Data Collection** - 설문, 데이터 수집
3. **Evaluation** - 성과 분석, KPI 평가
4. **Knowledge Transfer** - 학습 사항 문서화

**산출물**:
- Post-Event Report (PER)
- 비즈니스 인텔리전스
- Lessons Learned 문서

> "이벤트는 철거(move-out)가 완료될 때까지 끝나지 않습니다."

## 2. Processes (프로세스)

순차적이면서도 반복적인 시스템으로, 동적인 이벤트 특성에 대응합니다.

### 5가지 프로세스

| 프로세스 | 설명 | 주요 활동 |
|---------|------|----------|
| **Assess** | 평가/식별 | 요구사항 파악, 리스크 식별, 기회 분석 |
| **Select** | 선택 | 옵션 비교, 의사결정, 자원 할당 |
| **Monitor** | 모니터링 | 진행 상황 추적, 성과 측정, 편차 분석 |
| **Integrate** | 통합 | 요소 간 조정, 커뮤니케이션, 변경 관리 |
| **Document** | 문서화 | 기록 생성, 정보 보관, 증빙 수집 |

### 프로세스 흐름

```
    ┌──────────┐
    │  ASSESS  │ ←──────────────────────────────┐
    │   평가   │                                │
    └────┬─────┘                                │
         ↓                                      │
    ┌──────────┐                                │
    │  SELECT  │                                │
    │   선택   │                                │
    └────┬─────┘                                │
         ↓                                      │
    ┌──────────┐                                │
    │ MONITOR  │ ─────→ 피드백 ─────────────────┤
    │ 모니터링 │                                │
    └────┬─────┘                                │
         ↓                                      │
    ┌──────────┐                                │
    │INTEGRATE │ ─────→ 조정 필요 시 ───────────┘
    │   통합   │
    └────┬─────┘
         ↓
    ┌──────────┐
    │ DOCUMENT │
    │  문서화  │
    └──────────┘
```

### 2.1 Assess (평가)

**목적**: 필요사항, 문제, 기회 식별

**도구 및 기법**:
- 스태프 미팅
- 이해관계자/벤더 인터뷰
- 체크리스트
- 브레인스토밍
- 마인드맵
- 이전 이벤트 문서 검토

**활동**:
- 기획/실행 단계에서 전술 수립
- 이벤트 중 발생 문제 해결
- 추가 필요사항 식별

### 2.2 Select (선택)

**목적**: 최적 대안 선택 및 자원 배분

**활동**:
- 옵션 비교 분석
- 비용-효과 분석
- 공급업체 선정
- 장소 선정
- 프로그램 결정

### 2.3 Monitor (모니터링)

**목적**: 진행 상황 추적 및 편차 관리

**활동**:
- 예산 vs 실제 비용 추적
- 일정 준수 여부 확인
- 등록/판매 현황 모니터링
- 품질 관리
- 리스크 모니터링

### 2.4 Integrate (통합)

**목적**: 모든 요소의 원활한 조정

**Note**: 국제 EMBOK 모델에서는 "Communicate"로 표현

**활동**:
- 부서 간 조정
- 이해관계자 커뮤니케이션
- 변경 사항 통합
- 충돌 해결
- 자원 재배분

### 2.5 Document (문서화)

**목적**: 활동 및 결정 기록

**활동**:

| 활동 | 설명 |
|------|------|
| 기록 생성/유지 | 활동 및 결정 사항 기록 |
| 평가 지원 정보 | 분석을 위한 데이터 수집 |
| 증빙 자료 수집 | 검증 및 확인용 문서 |
| 비즈니스 정보 | 아카이브용 자료 정리 |

## Phases × Processes 매트릭스

각 단계(Phase)에서 모든 프로세스(Process)가 적용됩니다:

|  | Initiation | Planning | Implementation | Event | Closure |
|--|------------|----------|----------------|-------|---------|
| **Assess** | 타당성 평가 | 요구사항 분석 | 준비상태 점검 | 실시간 상황 평가 | 성과 평가 |
| **Select** | 컨셉 선택 | 공급업체 선정 | 최종 확정 | 즉석 결정 | 우수사례 선정 |
| **Monitor** | 범위 모니터링 | 예산/일정 추적 | 진행상황 추적 | 현장 모니터링 | 결과 측정 |
| **Integrate** | 팀 구성 | 계획 통합 | 실행 조정 | 현장 조정 | 피드백 통합 |
| **Document** | 헌장 작성 | 계획서 작성 | 진행 보고 | 현장 기록 | 최종 보고서 |

## 핵심 원칙

1. **순차적이면서 반복적**: 선형적 진행이지만 필요시 이전 단계로 돌아감
2. **동적 접근**: 변화하는 상황에 적응
3. **지속적 커뮤니케이션**: 모든 단계에서 소통 필수
4. **사전 예방적**: 문제 발생 전 예측 및 대비
5. **순환적**: 다음 이벤트를 위한 피드백 루프
